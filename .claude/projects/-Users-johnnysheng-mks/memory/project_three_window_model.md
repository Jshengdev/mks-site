Three-window orchestration model established 2026-03-14:

- **Window 1 (Build):** Environment Worlds — execute multi-world routing with vastly different visual styles, push engine limits
- **Window 2 (Research):** Creative Coding AutoResearch — adapted from Karpathy's autoresearch, autonomous pipeline that experiments with shader techniques, atmosphere values, visual approaches
- **Window 3 (Think):** Second Brain — the user + Claude talking, monitoring W1 and W2, enforcing depth over breadth, creative direction

**Why:** The loop: W3 seeds direction → W2 researches techniques → W1 builds them → W3 reviews and redirects.

**How to apply:** W3 (this conversation pattern) is always the starting point. Never launch build work without creative direction. The user's core struggle is depth vs breadth — AI defaults to going wide, but craft lives in going narrow.

**Key insight from user:** "It's very easy to go wide and get a lot of base pieces done, but it's very difficult to go narrow and find exact references to copy and build something like this."
